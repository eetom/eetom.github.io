.gitignore and the dreaded .DS_store

Last Updated: 10/31/2017

If you ever use a Mac for your web developing needs, you might see a certain file that shows up from time to time. The .DS_Store file is a file created by the Mac operating system that stores custom attributes of the folder kinda like a metadata of what has been changed about that folder. The problem is that this becomes annoying when using a version control process (such as Git) that you will have to constantly add and commit this even when you have made no code change you might have .DS_Store changes to update.

The solution is in the .gitignore file!

The .gitignore file is primarily used for Git version control to igonre files that the user does not want it to track changes. This makes it perfect for the use of managing the dreaded .DS_Store files (yes files, each folder will have it's own .DS_Store file)

Step 1: Create a .gitignore file

The first thing to do will be to create the .gitignore file

Check out a previous post on Command line basics for Designers for some help with command line to generate a .gitignore file.

Start in the git repository folder you wish to create the .gitignore file in. Then type in...

User$ touch .gitignore

Now you have an empty .gitignore file

Step 2: Let the .gitignore file ignore .DS_Store files

Now that we have a .gitignore file we can start telling Git what to ignore! Open up the .gitignore file in your favorite IDE/text editor and add in ...

.DS_Store

And then save. Then you're done! Well, almost...?

Step 3 Optional: Untracking tracked .DS_Store files

Just setting up the .gitignore is not enough if you have already started to track .DS_Store in a previous commit. You have to stop git from tracking it for .gitignore to take effect. Go back to terminal and the directory and type in ...

User$ git rm --cached .DS_Store

Step 4 Optional: The Nuclear option - Global .gitignore

That's right, we're sick and tired of having to do this for every repository we create. It's time once and for all to show .DS_Store who's boss. Github help files suggest using this line in your terminal to create a global .gitignore...

User$ git config --global core.excludesfile ~/.gitignore_global

And then once again, open up the file in a text editor/IDE and add the line...

.DS_Store

This should take care of your .DS_Store issues for good! Make sure to untrack (Step 3 optional) any .DS_Store files that are already being tracked when you do this as well!
